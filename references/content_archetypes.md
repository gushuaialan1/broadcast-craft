# 口播内容原型 / Broadcast Archetypes

> 本文档继承 khazix-writer 的「五种文章原型」，重构为适配短视频口播的五种内容原型。
> 每种原型包含：核心口证、话术结构、情绪曲线、时长配比、平台适配、专项要求。

---

## 1. 调查实验型

> **原文引用（khazix-writer）：** 核心口证是"我替你去做了这件事"。

### 口播化定义
"我亲自去做了，结果让我...”

### 话术结构
```
[0-3s]   反常识钩子："你以为这个东西很简单？我试了一下，发现..."
[3-8s]   建立信任："我花了X天/X万，亲自测试了这个..."
[8-20s]  实验过程（快节奏切片）
[20-35s] 第一个发现/踩坑
[35-50s] 第二个发现/反转
[50-55s] 结论归纳
[55-60s] 回环呼应 + CTA
```

### 情绪曲线
好奇 → 认真 → 困惑/崩溃 → 惊喜 → 温暖

### 时长配比
- 30s：不适用。调查实验需要展示过程，至少60s。
- 60s：精简版，只展示1个核心发现。
- 180s：完整版，展示2-3个发现 + 升维。

### 平台适配
- 抖韹★★★★★（极客类内容热门）
- B站★★★★★（完播率高）
- 快手★★★☆☆
- 视频号★★★☆☆

### 专项要求
- 必须有**真实的过程**：时间、地点、花费、遇到的困难
- 必须有**视觉证据**：屏幕录制、现场拍摄、数据截图
- 必须有**对比**：期望 vs 现实
- **禁止**："我打听到"——必须是亲身经历

## 2. 产品体验型

> **原文引用（khazix-writer）：** 核心口证是"跟我一起玩"。

### 口播化定义
"我给你看个好东西，保证你没见过"

### 话术结构
```
[0-3s]   好奇钩子："我发现了一个XXX，真的太疯了"
[3-8s]   场景建立："今天我在XXX的时候，偶然发现了..."
[8-18s]  展示过程："我给你看看怎么用"【操作屏幕】
[18-28s] 惊艳效果："你看，这是结果"
[28-45s] 实际应用："那么这个能干什么呢？我试了几个场景..."
[45-55s] 诊断/局限性："当然它也不是完美的，有几个缺点我得说..."
[55-60s] 总结 + 获取方式 + CTA
```

### 情绪曲线
好奇 → 轻松 → 惊喜 → 兴奋 → 真诚

### 时长配比
- 30s：只展示1个核心功能，适合简单工具。
- 60s：展示2个功能点 + 1个场景。
- 180s：深度体验，展示3-4个场景 + 对比评测。

### 平台适配
- 抖韹★★★★★
- 快手★★★★★
- B站★★★★☆
- 视频号★★★☆☆

### 专项要求
- 必须有**亲手操作**：不是"介绍功能"，是"我用给你看"
- 必须有**视觉展示**：屏幕录制或现场操作
- 必须有**惊艳效果**：要有"哦起"的时刻
- 必须有**局限性**：不能只说好不说坏

## 3. 现象解读型

> **原文引用（khazix-writer）：** 核心口证是"你注意到了吗？背后是什么？"。

### 口播化定义
"最近有个事儿，我觉得挺有意思的..."

### 话术结构
```
[0-3s]   现象钩子："最近有件事儿在网上很火，但我觉得大家都看偏了"
[3-8s]   建立共鸣："你是不是也刷到过？"
[8-20s]  现象描述（带视觉素材）
[20-35s] 浅层解读（“大家都在说...”）
[35-50s] 深层解读（“但是我觉得...”）
[50-55s] 文化升维/哲学连接
[55-60s] 回环呼应 + 留言引导
```

### 情绪曲线
好奇 → 认同 → 思考 → 顿悟 → 温暖/感慨

### 时长配比
- 30s：只适合“现象叙述”，不能深度解读。
- 60s：精简解读，1个观点。
- 180s：深度解读，3层刻画 + 升维。

### 平台适配
- 视频号★★★★★（转发率最高）
- B站★★★★★
- 抖韹★★★★☆
- 快手★★★☆☆

### 专项要求
- 必须有**视觉证据**：截图、录屏、现场拍摄
- 必须有**“但是”**：不能只是复述众说，必须有独立观点
- 必须有**升维**：从具体事件到普遍性命题
- 必须有**“我觉得”**：不能只是陈述事实，必须有主观判断

## 4. 工具分享型

> **原文引用（khazix-writer）：** 核心口证是"我发现了一个好东西"。

### 口播化定义
"有个工具/方法/小众，我自己用了很久，今天分享给你们"

### 话术结构
```
[0-3s]   干货钩子："不花一分钱，你就能..."
[3-8s]   个人故事："我之前一直被XXX问题困扰，直到我发现了..."
[8-20s]  工具展示：核心功能演示
[20-35s] 效果对比：用之前 vs 用之后
[35-50s] 使用场景：“其实不光是XXX，你还可以用它来..."
[50-55s] 局限性/注意事项
[55-60s] 获取方式 + CTA
```

### 情绪曲线
困扰 → 希望 → 兴奋 → 认真 → 温暖

### 时长配比
- 30s：不适用。工具分享需要展示过程，至少60s。
- 60s：精简版，1个核心功能。
- 180s：深度版，3-4个场景 + 使用技巧。

### 平台适配
- 快手★★★★★
- 视频号★★★★★
- 抖韹★★★★☆
- B站★★★☆☆

### 专项要求
- 必须有**个人故事铺垫**：不能直接介绍功能
- 必须有**效果对比**：用之前和用之后的差异
- 必须有**多场景**：不能只说一个用途
- 必须有**获取方式**：告诉用户怎么找到这个工具

---

## 5. 方法论分享型

> **原文引用（khazix-writer）：** 核心口证是"我把压箱底的东西掏给你了"。

### 口播化定义
"这个方法我用了X年，今天全部教给你们"

### 话术结构
```
[0-3s]   失败钩子："我之前用了X年才搞明白的一件事，今天一句话让你们懂。"
[3-8s]   谦逊铺垫："当然我也不是什么大佬，就是跳过了很多坑..."
[8-20s]  方法论核心：第一个原则/步骤
[20-35s] 第二个原则/步骤
[35-50s] 第三个原则/步骤
[50-55s] 学习曲线/踩坑警示
[55-60s] 总结 + 创作者声明 + CTA
```

### 情绪曲线
疑惑 → 谦虚 → 认真 → 希望 → 温暖

### 时长配比
- 30s：只能分享1个原则，不建议。
- 60s：精简版，1-2个核心原则。
- 180s：完整版，3-4个原则 + 案例。

### 平台适配
- 视频号★★★★★
- B站★★★★★
- 快手★★★★☆
- 抖韹★★★☆☆

### 专项要求
- 必须有**谦逊铺垫**：不能是"教你做人"，必须是"我跳过了很多坑"
- 必须有**可执行性**：用户看完知道"第一步做什么"
- 必须有**学习曲线**："我也是花了X年才懂的"，让用户有合理期望
- 必须有**案例**：抽象方法必须落到具体人/具体事

## 附录：原型选择决策树

```
内容是否包含"我亲自去做"的经历？
  是 → 调查实验型
  否 → 内容是否在展示一个具体产品/工具？
      是 → 内容是否以"我用给你看"为主？
          是 → 产品体验型
          否 → 工具分享型
      否 → 内容是否在解读一个公众现象/事件？
          是 → 现象解读型
          否 → 方法论分享型
```
