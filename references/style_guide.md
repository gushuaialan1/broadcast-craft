# 口播风格指南 / Broadcast Style Guide

> 本文档继承 khazix-writer 的「风格操作系统」，将其从"公众号长文"重构为"短视频口播"。
> 所有引用内容均来自 khazix-writer，适配部分标注明。

## 1. 节奏控制系统

> **原文引用（khazix-writer）：**
> 
> 卡兹克的节奏感，本质在于：长短句交替，单句成段。
> 
> 好的节奏是"波动"：围绕主线偏出去一点点，用一句话拉回来继续推进。最不要做的就是偏出去很远，然后硬拉回来，读者的心流会断掉。
> 
> 要绕着主线，勿作广泛闲聊，让读者觉得你在东一爪子西一棒槌。

### 1.1 口播特化节奏法则

口播比长文更极端：**6秒必须有一个情绪点**。

| 元素 | 长文标准 | 口播标准 | 原因 |
|---|---|---|---|
| 句子长度变化 | 长短句交替 | 极短句主导，偶尔一个长句推进 | 听觉波形需要更密 |
| 单句成段 | 至少3次 | 至少6次 | 每段一个“掌声” |
| 扣主线频率 | 每段落拉回一次 | 每15秒内必须有一句“定音针” | 注意力存储极端有限 |
| 停顿感 | 短句停顿 | **沉默也是节奏**（3秒沉默 = 强调） | 视频有画面，无声胜有声 |

### 1.2 口播节奏模板

以下是可复制的口播节奏模板，按时长分类：

**【30秒口播】**
```
[0-3s]  短句钩子（一句话）
[3-8s]  扩展惊讶（两句，比钩子稍长）
[8-15s] 信息打底（一句重量级信息）
[15-20s]反转/升维（短句炸弹）
[20-30s]回环呼应（回到钩子，一句收尾）
```

**【60秒口播】**
```
[0-3s]   短句钩子
[3-8s]   建立共鸣（“你是不是也…”）
[8-18s]  第一个信息高峰（三句快节奏）
[18-25s] 过渡段（一句轻松的话）
[25-40s] 第二个信息高峰（感情浓度更高）
[40-50s] 文化升维或情绪峰值
[50-60s] 回环呼应 + CTA（关注/评论/点赞）
```

**【3分钟口播】**
每60秒一个“记忆定弹”（金句、重点、情绪点），共计3个。

## 2. 口语化表达

> **原文引用（khazix-writer）：**
> 
> 采用极其口语化的表达。量身定制的口癖、脏话、骨骼清奇的断句，让文章氛围轻松自然。
> 
> 这里的口语化是真正的“说话”，不是将书面语修改为口语化，而是从一开始就是用“说话”的方式思考。
> 
> 如果你写的话看不出来是书面语还是口语，那就是书面语。只有当你看到一句话能直接想象出“这个人在哪里说话”时，才是口语。

### 2.1 口播特化口语化

口播比长文更要"像说话"，因为用户**真的在听你说**。

**三重口语化检验**：
1. **默读测试**：把文案念出来，如果觉得脾腩不自在，就是AI写的。
2. **对着镜子说**：如果说出来会觉得傻，那就改到不傻。
3. **发给朋友看**：如果他觉得"这是你写的文章还是你说的话？"，回答必须是"说的话"。

### 2.2 口播常用口语结构

| 书面语 | 口播语 | 示例 |
|---|---|---|
| 当下 | 现在 | "现在很多人都在说…" |
| 其实 | 说实话 | "说实话我之前也这么想过" |
| 然而 | 但是 | "但是朋友们，事情不是这样的" |
| 因此 | 所以啊 | "所以啊，我今天必须得跟你们掳开说" |
| 值得注意的是 | 你发现没 | "你发现没，这里有个很细思极恐的东西" |
| 从某种程度上来说 | 得多少 | "得多少？我给你个数" |
| 据调查显示 | 我问过一个做这行的朋友 | 用人称代替数据 |

### 2.3 论述故意打断（引用+ 特化）

> **原文引用（khazix-writer）：**
> 
> 在逻辑论述中故意加入口语打断，破坏严谨性。让文章有了"温度"。
> 
> 形式：
> - 重复强调："就是...就是...就是单纯的"
> - 中途模糊："我就不说是谁了"
> - 省略打断："我寻思了一下我没寻思明白"
> - 情绪断裂："尼玛。"、"黑暗森林。"

**口播特化**：打断在口播中变成"纯音打断"——不是语言打断，是**沉默**。

| 打断类型 | 口播表现 | 效果 |
|---|---|---|
| 重复强调 | "就是...就是...就是单纯的" | 气势叠加 |
| 中途模糊 | "我就不说是谁了" | 制造親密感 |
| 情绪断裂 | 说完“尼玛”后**停3秒** | 沉默是最大的龄面 |
| 自我吐槽 | "我这么说可能会被骂吧" | 降低防御 |
| 装傻 | "我也不知道我说清楚没有" | 弱化教学感 |

## 3. 知识输出的"随手掏"模式

> **原文引用（khazix-writer）：**
> 
> 错误方式：“下面我来简单介绍一下格式塔心理学。格式塔心理学起源于20世纪初的德国…”
> 
> 正确方式：“突然想起了以前做交互设计的时候，有一个几乎刻在我血液里的心理学，这玩意，叫格式塔心理学。”
> 
> 差异在于：错误方式是**教学姿态**（我来教你），正确方式是**联想姿态**（聊着聊着想到了）。

**口播特化**：口播中的知识输出必须在 **10秒内**完成"问题感受 → 触发知识 → 答案解决"的闭环。

口播版“随手掏”结构：
```
"有没有人和我一样...？"（问题感受，2秒）
"我也是前两天才知道，这玩意叫XXX。"（知识触发，3秒）
"它最疯的一点是...，你细品。"（答案解决，5秒）
```

## 4. 人物画像法（引用+ 特化）

> **原文引用（khazix-writer）：**
> 
> 从一个数据点想象出一个完整的人，标准结构是：
> 1. 触发点数据（“1000个已付款”）
> 2. 快速代入（“他可能是一个...”）
> 3. 多维堆砌（城市→职位→生活→心理→具体行为）
> 4. 情感锚定（为什么这么做）
> 5. 细节具象化（十几平米出租屋、拼好饭）
> 
> **约束条件**：必须在3-5句话内让这个人物立体。

**口播特化**：口播中的人物画像必须在 **5秒内**完成立体。

快速人物画像公式：
```
"你认识这种人吗？"（问句，1秒）
"上次我去咖啡店，遇到一姐们，35岁，做运营的，月薪1万5，租在离公司三站路的老破小里，她跟我说..."（4秒）
```

## 5. 文化升维

> **原文引用（khazix-writer）：**
> 
> 不是硬凑升华，而是“聊着聊着自然想到了”。
> 
> 具体模式：
> - 具体技术话题 → 历史类比（AI安装费 → 1880年代电力普及）
> - 具体现象 → 哲学概念（AI看不到爱心 → “我们活在流中，AI活在帧中”）
> - 具体事件 → 文学意象（9.9元DeepSeek → 《北京折叠》）
> 
> 连接原则：升维必须是有机的。

**口播特化**：口播的升维必须在最后 **5-10秒**，且必须用 **短句** 完成。

口播升维结构：
```
"所以你看，"（停顿）
"很多事，从来都不是技术问题。"（1句）
"是人的问题。"（1句，停顿）
```

## 6. 回环呼应

> **原文引用（khazix-writer）：**
> 
> 编剧理论的直接移用：
> - 第一幕挂一把枪在墙上 → 第三幕必须开火
> - 前面埋的每一个细节后面都得“响”
> - 跨文章的签名式呼应

**口播特化**：口播的回环必须在最后 **3秒内**完成，且必须是**语音回环**（听起来有韵律）。

口播回环公式：
- **词语回环**：开头用的词，结尾再说一遍。例：开头"你知道吗，最恐怖的不是失败"，结尾"所以记住，最恐怖的不是失败，是你连试都没试过。"
- **场景回环**：开头的场景，结尾再回到那个场景。
- **CTA 回环**：回环后紧跟一个行动号召。

## 7. 绝对禁区（直接引用 khazix-writer）

> **原文引用：**
> 
> ### 7.1 格式性形式的AI味
> - 首先…其次…最后
> - 综上所述
> - 总的来说
> - 值得注意的是
> - 这意味着
> - 说白了
> - 本质上
> - 毫无疑问
> - 当今AI快速发展的时代…（超级踩雷）
> - 随着科技的进步…（AI感爆棚）
> - 这不仅仅是…更是…（升华模板）
> - 这不仅让我们看到了…也让我们思考…（双重升华）
> - 当今这个充满机遇与挑战的时代…
> 
> ### 7.2 标点禁令
> **严禁使用以下标点：**
> - 冒号（：）
> - 破折号（——）
> - 双引号（“”）
> 
> **允许使用：**
> - 逗号（，）
> - 句号（。）
> - 问号（？）
> - 感叹号（！）
> - 省略号（……）

### 7.3 口播增补禁区

**短视频特有的AI味**：
- "家人们谁懂啊"——极端口语化到虚假
- "天塌了"——情绪模板
- "是谁还不知道"——虚假亲密感
- "不是吧不是吧"——反复套路
- "我真的会谢"——网络口头禁区
- "所以说"——AI 最爱的过渡词
- "真的"——每句话带两个真的就是AI
- "其实呢"——文化升维必用词，但AI用得过度
- "让我们一起来看看"——新闻联播词

**平台调性禁区**：
- 抖音：禁止慢热跟风、禁止正经教学、禁止"谁懂啊"式装痛苦
- 快手：禁止装精英、禁止假大空、禁止过度剪辑感
- 视频号：禁止太吵闹、禁止纯娱乐、禁止过度演戏化

## 8. 口播格式规范

文案输出必须使用以下标准格式，方便演员/达人直接使用：

```markdown
## 文案标题
**时长**：XX秒
**平台**：抖音/快手/视频号/B站
**人设**：具体身份标签
**核心情绪**：愤怒/温暖/好奇/自嘲
**原型**：哪一种口播原型

### 口播稿（含时间轴）
[0-3s]  钩子
[3-8s]  承诺收益
...

### 注意事项
- 重音字标注：**粗体**
- 沉默点标注：【停2秒】
- 语气加强标注：【重音】
- 动作提示：【摇头】【笑】【指镜头】

### 预估情绪曲线
- 起：XX
- 承：XX
- 转：XX
- 合：XX
```
