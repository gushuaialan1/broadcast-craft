# Book-Bridge 方法论：史桥书单（用历史故事卖书）

## 核心定位

不是书评，不是读书总结，而是**用历史故事作为钩子，激活用户痛点，让书籍成为解决方案的自然出口**。

结构公式：
**历史困境（共鸣）→ 现代映照（扎心）→ 书中解法（希望）→ 行动召唤（转化）**

---

## 设计前提：零预设人群

`--target-persona` 是**完全自由的文本输入**，系统不预设任何人群分类。

- LLM 必须根据用户提供的原始描述**自主推导**痛点，不依赖任何预设模板
- 无论输入是"60岁退休老人，怕被社会遗忘"还是"16岁高中生，沉迷游戏不想学习"，系统都平等工作
- 方法论中的人群示例仅用于说明格式，**不是限制清单**

---

## 5 种史桥模型

| 模型 | 逻辑 | 示例（不限于职场） |
|:---|:---|:---|
| **镜鉴模型** | 历史人物困境 = 用户困境 | 用项羽刚愎自用 → 推《非暴力沟通》给管理层 |
| **因果模型** | 历史因果链 = 书中方法论原理 | 用诸葛亮事必躬亲 → 推《授权》给创业者 |
| **反差模型** | "如果用了书中方法，历史会怎样" | 用崇祯勤政亡国 → 推《高效能人士的七个习惯》给创业者 |
| **隐喻模型** | 历史典故 = 书籍概念隐喻 | 用"萧何月下追韩信" → 推《人才引力》给 HR |
| **弧光模型** | 历史人物成长 = 读者成长路径 | 用刘邦从混混到帝王 → 推《终身成长》给职场新人 |

## 5-Stage 渐进式工作流

### Stage 01 | Book Anchor（书籍锚定）

**目标**：锁定书籍核心价值主张，确认与历史题材的连接点。

**关键动作**：
1. 提取书籍核心论点（1 句话）
2. 确认书中是否有历史案例引用（如有，优先使用）
3. 确定书籍能解决的最深层心理需求（非表层功能）
4. **Fallback 处理**：若网上检索不到该书，使用 `--fallback-theme` 作为主题替代，明确标注

**产出**：书籍锚定卡

### Stage 02 | Pain Mapping（痛点映射）

**目标**：将 `--target-persona` 的原始描述转化为结构化痛点分析。

**核心原则**：
- **零人群预设**：LLM 必须基于原始文本自主推导，不能套用"中层管理者""创业者"等模板
- **多维度深挖**：从描述中识别身份、场景、情绪、渴望四个维度
- **历史类比**：找到一个历史人物曾经历过的相似困境

**产出**：痛点映射卡 + 历史类比人物

### Stage 03 | Story Bridge（故事搭桥）

**目标**：选定一个历史故事，让它同时承担三个功能：
1. **钩子**：3 秒抓住注意力
2. **共鸣**：让观众觉得"我也是"
3. **桥梁**：自然过渡到书籍

**故事选取标准**：
- 必须是具体场景，不是抽象评价
- 必须有情绪高潮点（愤怒、恐惧、遗憾、释然）
- 必须能在结尾处与书籍形成对照

**产出**：故事搭桥卡 + 情绪曲线设计

### Stage 04 | Compose（成稿）

**目标**：输出 300-450 字口播文案，含语气标注。

**结构模板**：
```
[开场钩子]        历史场景切入（3秒）
[故事展开]        具体细节，制造沉浸感（20秒）
[转折扎心]        "但你知道吗？XXXXX 今天还在发生"（10秒）
[痛点放大]        描述用户正在经历的具体场景（15秒）
[书籍登场]        "有一本书，讲的就是这个……"（10秒）
[方案对照]        书中方法 vs 历史人物的失败选择（20秒）
[CTA]            "如果你也……，值得看看"（5秒）
```

**语气标注**：
- **粗体** = 重音
- 【停顿】= 停顿 1 秒
- 【放慢】= 语速放缓
- 【加速】= 加快节奏

**产出**：完整口播稿 + 情绪曲线

### Stage 05 | Review（软植入质检）

五问审查：
1. **价值感优先**：前 60% 是否能独立作为历史短视频观看？
2. **不硬广**：书名是否在视频后半段才出现？
3. **人群精准**：如果删除书名，目标用户是否依然觉得"这说的就是我"？
4. **不夸大**：是否有"读完就能改变人生"的过度承诺？
5. **Fallback 合规**：若使用 fallback-theme，是否明确标注了"围绕 XX 主题展开"？

**评级**：
- **PASS** = 可直接输出
- **NEEDS_REVISION** = 需修改后重新进入 Stage 04

---

## 软植入红线

| 检查项 | 标准 |
|:---|:---|
| **价值感优先** | 历史故事本身必须有独立观看价值，不能是书籍的附庸 |
| **不硬广** | 前 60% 不得出现书名，只能出现"有一位古人/有一本书" |
| **不夸大** | 不得声称"读完这本书就能……"，只能用"提供了一种视角" |
| **CTA 自然** | 结尾引导购书必须基于"如果你也对 XX 感到困惑" |
| **Fallback 合规** | 若使用 `--fallback-theme`，必须在文案中标注"围绕 XX 主题展开" |

## 状态文件设计

```
book_title              书名
fallback_theme          备选主题（当检索不到书时使用）
target_persona          用户画像原始文本
pain_identity           身份标签
pain_scenario           生活场景
pain_emotion            核心情绪
pain_aspiration         渴望状态
historical_analog       历史类比人物
story_bridge            选定的历史故事
script_draft            口播文案草稿
review_status           PASS / NEEDS_REVISION
```
