# Stage 04 | Compose（成稿）

## 输入

- 书籍锚定：stage_01_book_anchor.json
- 痛点映射：stage_02_pain_mapping.json
- 故事搭桥：stage_03_story_bridge.json
- 时长：{{length}}
- 平台：{{platform}}

## 任务

1. 根据情绪曲线输出 300-450 字口播文案
2. 加入语气标注：**粗体**、【停顿】、【放慢】、【加速】
3. 书名必须在后半段出现
4. CTA 自然，基于用户痛点而非書籍功能

## 结构模板

```
[开场钩子]        历史场景切入（3秒）
[故事展开]        具体细节，制造沉浸感（20秒）
[转折扎心]        "但你知道吗？XXXXX 今天还在发生"（10秒）
[痛点放大]        描述用户正在经历的具体场景（15秒）
[书籍登场]        "有一本书，讲的就是这个……"（10秒）
[方案对照]        书中方法 vs 历史人物的失败选择（20秒）
[CTA]            "如果你也……，值得看看"（5秒）
```

## 输出格式（JSON）

```json
{
  "script": "完整口播稿文本，含语气标注",
  "estimated_duration": "预估时长，如 90s",
  "hook_line": "开场钩子（前 3 秒突出显示）",
  "cta_line": "行动召唤语",
  "platform_notes": "针对 {{platform}} 的优化建议"
}
```
